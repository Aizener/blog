（转）关于nrm和npx两个工具的使用
发表于 2020-12-30| 更新于 2020-12-30|Web前端
阅读量:

这里介绍几个关于npm相关的几个好用的工具，其中有：nrm，npx；这两个工具都简单实用，让我们使用npm更加的方便安逸了。

nrm

首先来说说nrm，这是一款npm的资源管理器，允许我们快速的切换源，这样就省的了来回切换源输入繁琐的命令拉，什么！你问我为啥要切换源？比如，你发布一个npm包，就得切换成官方的源咯。

安装

这里列举出常用命令：

  • nrm ls 列出可用的源

  • nrm use taobao 选择国内淘宝的源

  • nrm add taobao http://192.168.10.127:8081/repository/npm-public/添加源

  • nrm del taobao删除对应的源

  • nrm test npm 测试速度

这些命令已经足够用来使用nrm了，毕竟我们只是为了更方便快捷的切换不同的源，仅此而已。

npx 的原理很简单，就是运行的时候，会到node_modules/.bin路径和环境变量$PATH里面，检查命令是否存在。

由于 npx 会检查环境变量$PATH，所以系统命令也可以调用。

注意，Bash 内置的命令不在$PATH里面，所以不能用。比如，cd是 Bash 命令，因此就不能用npx cd。

–no-install参数和–ignore-existing` 参数

如果想让 npx 强制使用本地模块，不下载远程模块，可以使用--no-install参数。如果本地不存在该模块，就会报错。

反过来，如果忽略本地的同名模块，强制安装使用远程模块，可以使用--ignore-existing参数。比如，本地已经全局安装了create-react-app，但还是想使用远程模块，就用这个参数。
